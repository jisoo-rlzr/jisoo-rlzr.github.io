MacOS에서 adb(Android Debug Bridge) 사용 설정하기

안드로이드 개발을 한다면 필수적으로 사용하는 adb를 사용 설정하는 법을 정리하겠다.


0. 사전 준비

Android Studio 설치 및 환경 설정이 완료된 상태로 가정한다.

1. adb 설치하기

adb는 Android SDK Manager Platform-Tools 패키지에 포함되어 있다.
이는 Android Studio > SDK Manager 실행 > SDK Tools 탭에서 다운로드 할 수 있다.

2. Path 환경변수 설정하기

adb를 사용하기 위한 마지막 단계는 환경변수를 설정하는 것이다.

  1. .bash_profile 파일 열기.
$ vi ~/.bash_profile
  1. ADB_HOME 변수 추가하기.
    Android SDK 경로는 Android Studio SDK Manager 에서 확인할 수 있다.
export ADB_HOME=[Android SDK 경로]/platform-tools
  1. ADB_HOME 변수를 PATH에 추가하기.
export PATH=$PATH:$ADB_HOME
  • 최종적으로 나의 .bash_profile에서 Android 관련 설정 내용은 다음과 같다.
export ANDROID_HOME=$HOME/Library/Android/sdk
export ADB_HOME=$ANDROID_HOME/platform-tools
export PATH=$PATH:$ANDROID_HOME/emulator
export PATH=$PATH:$ANDROID_HOME/tools
export PATH=$PATH:$ANDROID_HOME/tools/bin
export PATH=$PATH:$ADB_HOME
  1. .bash_profile 파일 저장 후 닫기.
  2. .bash_profile 파일 새로고침. 새로고침 해야 환경변수에 등록된다!
$ source ~/.bash_profile
  1. adb 명령어를 실행해 제대로 적용된 것을 확인한다.
$ adb version
